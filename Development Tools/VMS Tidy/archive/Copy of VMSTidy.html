Overview

VMS Tidy is designed to tidy up the mess of code that Visual Max Script leaves behind.

This makes for much quicker prototyping of tools as you can just jump in and start throwing UI elements around without worrying that VMS will f*ck it up after each edit.

VMS Tidy is an accompanying tool to Visual Max Script that lets you concentrate on the task of designing functional user interfaces, before tidying and re-ordering it's output code to match the visuals.

Getting started

VMS Tidy needs to have some rollout code to work with, so copy and paste some existing code from an editor window into the text box, or use the Sample Code included in the interface for a quick test.

Now, click "Tidy..." and watch the code magically re-arrange itself!

Interface elements are rearranged in the order that they are found in the interface – reading from left to right, top to bottom, with groups taking priority, and nesting controls found within them.

Adding manual line breaks

VMS Tidy is fairly strict about it's tidy-up, but there are times when you will want to manually start a new block.

To insert a line break before a rollout control just add a comment (--) to the end of the line.

Other controls

Click "Edit..." to open the newly tided code in a new (blank) rollout, so you can continue to add, edit, or move items easily, using the Visual Max Script editor.

Click "Test..." to create a new (exactly sized) rollout and check that everything has worked.

Use the "Sample Code" drop down to view and test sample layouts straight away. This is meant for learning purposes only.

Working methodology

It's up to you how you want to work with VMS Tidy. You can tidy existing code, or start a rollout from scratch by clicking "Edit...".

To start a new User Interface you may like to:

  • Create a new script by clicking "Edit..."
  • Open it in Visual Max Script and start designing visually
  • Once happy, name the controls logically
  • When finished, paste the script into VMS Tidy and tidy it
  • Re-output the code and start associating handlers and functions to rollout controls.

What VMS Tidy isn't

VMS Tidy is not a visual code designer, or a replacement for the Visual Max Script Editor.
