VMS Tidy

Overview

VMS Tidy is an accompanying tool to Visual MaxScript designed to tidy up the mess of code it leaves behind.

VMS Tidy lets you concentrate on the task of designing functional user interfaces within Visual MaxScript before tidying and re-ordering it's output to match the visuals.

This makes for much quicker prototyping of tools as you can just jump in and start throwing UI elements around without worrying about:

  • Visual relationships between UI elements and code
  • Relationships between groups and contained controls
  • Out-of order lines of code
  • Whitespace between blocks of code
  • Code indentation

What VMS Tidy isn't

VMS Tidy is not a visual code designer, or a replacement for the Visual MaxScript Editor.

Getting started

VMS Tidy needs to have some rollout code to work with, so copy and paste some existing code from an editor window into the text box.

Alternatively use the Sample Code drop down included in the interface to jump right in.

Now, click "Tidy..." and watch the code magically re-arrange itself!

Rules and formatting

VMS Tidy expects only rollout controls, not the entire rollout definition, variables or event handlers.

Controls are rearranged in the order that they are found in the interface – reading from left to right, top to bottom – with GroupBoxes taking priority, then nested controls found within them, and finally lone controls.

Any code that might potentially cause problems (such as variables within properties) will be commented-out and placed at the end for safety's sake.

Adding manual line breaks

VMS Tidy is fairly strict about it's tidy-up, but there are times when you will want to manually start a new block.

To insert a line break before a rollout control just add a comment (--) to the end of the line.

Testing your code

Click "Test..." to create a new (exactly sized) rollout and check that everything has worked.

Click "Edit..." to open the newly tided code in a new (blank) rollout, so you can continue to add, edit, or move items easily, using the Visual MaxScript editor.

Other tools

Use "C" and "P" to copy and paste code directly if you have the Avguard Extensions installed.

Use "<" and ">" to undo and redo edits or tidies.

Use "Renumber" to automatically rename/renumber controls with default names (eg. btn, ddl, chk).
A list of buttons, for example btn12, btn1, btn5 would be renamed to btn1, btn2, btn3.

To resize the dialog, use the drop down on the far-right.

Sample code

Play with any of the code examples in the "Sample Code" drop-down to get to know the program a little better before letting it loose on your own code!

Working methodology

It's up to you how you want to work with VMS Tidy.

You can tidy existing code, or create an empty rollout to use as a starting point by clicking "Edit...".

To start building a new UI layout you may like to:

  • Create a new script by clicking "Edit..."
  • Open it in Visual MaxScript and start designing
  • Name the controls, if you like
  • Once finished, paste the script back into VMS Tidy and click "Tidy..."
  • Re-output the code and start associating handlers and functions to rollout controls.
